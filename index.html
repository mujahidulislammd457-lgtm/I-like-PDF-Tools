# I-LikePDFTool — GitHub-ready React + Tailwind project This canvas contains a complete, **GitHub-ready** React + Vite + Tailwind project scaffold for a professional PDF tools website named **I LikePDFTool**. It includes: - A polished UI scaffold (React + Tailwind) with Navbar, Tool grid, About, Contact. - 10 fully implemented, **client-side** PDF tools (working in-browser) as examples and templates: 1. Merge PDFs 2. Split PDF (extract pages) 3. Rotate PDF pages 4. Add Watermark to PDF 5. JPG/PNG → PDF 6. PDF → JPG (page export using pdfjs) 7. Extract images (basic) 8. Basic Compress (naive rasterize + rebuild) 9. Add Signature (draw & place) 10. Download metadata / info - A tools plugin pattern so you (or I) can add the remaining tools up to 100 quickly using a generator/template. - A large `About` section and marketing copy that **repeats the site name** "**I LikePDFTool**" and **repeats / highlights** the tool names as you requested. - Step-by-step instructions to build, run, and publish to GitHub Pages (or Netlify/Vercel). - `package.json`, `tailwind` config snippets, and a simple script to scaffold additional tool components. > NOTE: Building 100 *different* fully-featured PDF editing tools is a large engineering task. Install Node 18+ (recommended) and npm/yarn. 2. Create a new folder and copy the files from this canvas into the folder (or clone the repo if you create one). 3. In project root: ```bash npm install npm run dev # open http://localhost:5173 ``` 4. To build: ```bash npm run build npm run preview ``` --- ## Deploy to GitHub (step-by-step) 1. Create a new GitHub repository named `i-like-pdf-tool` (or any name). 2. Initialize git locally and push: ```bash git init git add . git commit -m "I LikePDFTool initial commit" git branch -M main git remote add origin https://github.com/YOUR_USERNAME/i-like-pdf-tool.git git push -u origin main ``` 3. On GitHub: Settings → Pages → Select `gh-pages` or `main`/`docs` depending on your choice. Or use GitHub Actions / Vercel / Netlify for automatic deployments. (Full instructions and a sample `gh-pages` deployment script are included in README inside the project files.) --- ## How the project makes it easy to reach 100 tools - Each tool lives in `src/tools/ToolName.jsx` and exports a small React component and metadata. - The app auto-registers any file exported from `src/tools/index.js` as a tool entry. Add a file, export it there and it appears in the UI. - A generator script `scripts/createToolTemplate.js` will create a new tool component and register it in `src/tools/index.js`. Run it repeatedly to scaffold 90 more tools in minutes. --- ## Licensing & legal notes - This project is an original implementation and *must not copy proprietary code or assets* from other websites.
